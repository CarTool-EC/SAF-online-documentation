Glossary of Terms
-----------------

+---------------------+------------------------------------------------+
| Term                | Definition                                     |
+=====================+================================================+
| Architecture        | Abstract, reusable element that defines        |
| Building Block      | required capabilities and interoperability     |
| (ABB)               | characteristics without specifying             |
|                     | implementation.                                |
+---------------------+------------------------------------------------+
| Architecture        | Method that structures architecture activities |
| Implementation      | into phases aligned with analysis, design, and |
| Method (AIM)        | implementation lifecycle use cases.            |
+---------------------+------------------------------------------------+
| Architecture        | A rule or guideline governing the design and   |
| Principle           | use of IT resources and architecture           |
|                     | decisions.                                     |
+---------------------+------------------------------------------------+
| Cartography         | Structured mapping of interoperability         |
|                     | solutions and architecture elements to support |
|                     | reuse and decision-making.                     |
+---------------------+------------------------------------------------+
| Compliance          | Verification that a solution adheres to legal, |
|                     | regulatory, and policy requirements.           |
+---------------------+------------------------------------------------+
| Conformance         | Verification that an architecture or solution  |
|                     | correctly applies defined standards, building  |
|                     | blocks, and modelling rules.                   |
+---------------------+------------------------------------------------+
| Digital Business    | A capability required to support the delivery  |
| Capability (DBC)    | and operation of digital public services.      |
+---------------------+------------------------------------------------+
| Digital Public      | A service delivered by a public administration |
| Service (DPS)       | to citizens, businesses, or other              |
|                     | administrations using information and          |
|                     | communication technologies.                    |
+---------------------+------------------------------------------------+
| Increment           | A partial implementation that builds           |
|                     | cumulatively on previous implementations.      |
+---------------------+------------------------------------------------+
| Interoperability    | The ability of systems and organisations to    |
|                     | work together across legal, organisational,    |
|                     | semantic, and technical dimensions.            |
+---------------------+------------------------------------------------+
| Interoperability    | Platform for testing conformance of systems    |
| Test Bed            | against semantic and technical specifications. |
+---------------------+------------------------------------------------+
| Intervention        | A pilot activity focused on the definition and |
|                     | design of interoperable digital public         |
|                     | services.                                      |
+---------------------+------------------------------------------------+
| Legal               | Alignment of legislation, policies, and        |
| Interoperability    | regulatory frameworks enabling digital public  |
|                     | services to operate across jurisdictions.      |
+---------------------+------------------------------------------------+
| Organisational      | Alignment of processes, roles, and governance  |
| Interoperability    | structures to enable collaboration between     |
|                     | organisations.                                 |
+---------------------+------------------------------------------------+
| Reference           | A standardised architectural framework used to |
| Architecture (RA)   | guide the design and evaluation of systems.    |
+---------------------+------------------------------------------------+
| Semantic            | Shared understanding of data through common    |
| Interoperability    | vocabularies, data models, and definitions.    |
+---------------------+------------------------------------------------+
| Solution            | Framework that structures problem analysis and |
| Architecture        | solution design within the architecture        |
| Framework (SAF)     | lifecycle.                                     |
+---------------------+------------------------------------------------+
| Solution Building   | Concrete implementation of one or more ABBs    |
| Block (SBB)         | using specific technologies, standards, or     |
|                     | products.                                      |
+---------------------+------------------------------------------------+
| Technical           | Compatibility of systems, applications, and    |
| Interoperability    | infrastructure enabling communication and data |
|                     | exchange.                                      |
+---------------------+------------------------------------------------+
| Tranche             | A partial implementation that is largely       |
|                     | independent of previous implementations within |
|                     | a lifecycle.                                   |
+---------------------+------------------------------------------------+
| Validation          | Assessment that a solution meets functional,   |
|                     | non-functional, and interoperability           |
|                     | requirements.                                  |
+---------------------+------------------------------------------------+
